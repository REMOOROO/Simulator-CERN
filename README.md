# Симуляция Большого адронного коллайдера — пульт оператора (Unity)

Учебный проект: интерактивная симуляция работы ускорителя ЦЕРН **от лица
оператора за пультом управления**. Оператор выбирает сорта ионов для двух
встречных пучков, запускает цикл ускорителя и наблюдает соударения и
рождение частиц в детекторе.

## Что сталкивают на самом деле

На LHC сталкивают не атомы, а **ионы** — ядра атомов, полностью лишённые
электронов (а протон — это просто ядро водорода). В симуляции доступно
16 ядер, реально используемых в ускорителях мира (LHC, RHIC, SPS, FAIR, NICA):

p (протон), d (дейтрон), ⁴He, ¹²C, ¹⁶O, ²⁰Ne, ²⁸Si, ⁴⁰Ar, ⁴⁰Ca,
⁶³Cu, ⁸⁴Kr, ⁹⁶Ru, ¹²⁹Xe, ¹⁹⁷Au, ²⁰⁸Pb, ²³⁸U

Любую пару можно столкнуть друг с другом (включая асимметричные режимы
вроде p+Pb — такие сеансы на LHC тоже проводятся).

## Физика в модели

- **Энергия пучка** на нуклон определяется жёсткостью магнитов:
  E = 6,8 · (Z/A) ТэВ. Поэтому протоны получают 6,8 ТэВ, а свинец — только
  2,68 ТэВ/нуклон.
- **Энергия столкновения** √s(NN) = 2·√(E₁·E₂). Для Pb+Pb получается
  5,36 ТэВ — точное реальное значение сеансов LHC.
- **Множественность** рождённых частиц растёт с массой ядер: p+p даёт
  десятки частиц, Pb+Pb — сотни (в реальности тысячи).
- При столкновении тяжёлых ядер образуется **кварк-глюонная плазма** —
  состояние материи с температурой ~5·10¹² K, существовавшее в первые
  микросекунды после Большого взрыва (в симуляции — огненная вспышка).
- Треки **заряженных** частиц закручиваются полем детектора (r = p/qB),
  направление закрутки зависит от знака заряда; нейтральные летят прямо.
- **Осколки-спектаторы** — не участвовавшие в соударении части ядер —
  продолжают лететь вдоль направления пучка.
- В режиме p+p случайно выпадает событие **H → γγ** — кандидат в бозон
  Хиггса (отсылка к открытию 2012 года).

## Пульт оператора

Нижняя консоль управляется мышью:

- **Состав пучков** — стрелками `<` `>` выбирается ион для пучка 1 и пучка 2;
- **ПУСК** — цикл ускорителя: инжекция → разгон (с докладом процентов) →
  стабильные пучки → соударения;
- **СБРОС ПУЧКА** — вывод пучка в поглотитель, остановка сеанса;
- **ПОЛЕ ДЕТЕКТОРА** — вкл/выкл соленоид 3,8 Тл (без поля треки прямые);
- **РЕЖИМ АВТО/РУЧНОЙ** — непрерывные соударения или по одному;
- **Журнал** — лог операций с таймстампами, как в настоящем ЦУПе.

Дублирование с клавиатуры: `ПРОБЕЛ` — пуск, `X` — сброс, `B` — поле, `P` — авто.

Сверху — статусная панель: текущая энергия пучков, √s(NN), счётчик событий,
набранная светимость, состояние магнитов (1,9 K) и вакуума. Справа после
каждого соударения — сводка события со списком рождённых частиц.

## Как запустить

1. Unity 6 (проект создавался в 6000.4.x), шаблон **Universal 3D**.
2. Скопировать `Assets/Scripts/ColliderSimulation.cs` в папку `Assets`.
3. В пустой сцене: `GameObject → Create Empty`, повесить компонент
   `ColliderSimulation`.
4. Нажать **Play** — сцена и пульт строятся скриптом автоматически.

Поддерживаются обе системы ввода Unity (Input System и Input Manager).

## Условности модели

Масштабы и скорости приведены к наглядным (реальное кольцо — 26,7 км,
11 245 оборотов пучка в секунду, до 1 млрд соударений в секунду).
Качественная картина — цикл ускорителя, зависимость энергии от Z/A,
рост множественности с массой ядер, закрутка треков полем, КГП —
соответствует реальной физике.
